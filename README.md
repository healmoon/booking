# booking
**Данный проект представляет собой сайт по аренде коммерческой недвижимости большого и малого метража. 
Пользователь сайта может ознакомиться с имеющимися вариантами для аренды и забронировать необходимые.**

**Настройка проекта с нуля**
1. Открыть папку с вашими проектами на компьютере. Зажать ПКМ + CTRL. Открыть Терминал и ввести git clone https://github.com/*YOURREP*   - ввести ссылку на ваш репозиторий. Так вы скопируете репозиторий себе на компьютер;
2. Вводим в терминале python.exe для проверки доступности Пайтона. Если повяляется версия, то нажимаем quit(), если нет то установите python;
3. Вводим в терминале cd .\booking\ для входа в папку проектом;
4. Проверяем установку модуля virtualenv: вводим в терминал python -m venv -h  Если видите команды модуля venv, то модуль установлен. Если не увидели вводим python -m install virtualenv;
5. В терминале вызываем модуль venv  и в качестве первого параметра передаем ему имя директории с нашим виртуальным питоном. Общепринято называть также venv. После ввода в папке с проектом видим папку venv;
6. Для начала использования virtualenv нужно его активировать: вводим в терминале .\venv\Scripts\activate После чего видим в терминале слева (venv) зелеными буквами. С этого момента все команды, которые хоть как то связаны с пайтоном и запуском нашего проекта мы должны выполнять с запущенным virtualenv, чтобы не поставить виртуальными зависимости на системный пайтон;
7. Создаем в папке с проектом файл с зависимостями requirements.in Открываем этот файл и указываем  следующие зависимости, которые нужны для нашего проекта: Django==4.2.2 и djangorestframework;
8. Возращаемся в терминал, обновляем pip до последней версии puthon.exe -m pip install --upgrade pip;
9. Устанавливаем модуль pip tools, который в дальнейшем будет разрешать зависимости: pip install pip-tools;
10. Вводим в терминал команду pip-compile.exe который на вход берет файл requirements.in и компилирует его в requirements.txt В папке с проектом появится файл requirements.txt с зависимостями. В дальнейшем этот файл руками не правим, а только через requirements.in;
11. Устанавливаем зависимости: вводим в терминале pip-sync.exe;
12. Проверим установился ли джанго: python -m django --version после ввода этого в терминал должны увидеть версию джанго;
13. Теперь создаем проект: вводим в терминал django-admin startproject booking;
14. В папке с проектом появилась папка booking, переименовываем ее в booking1  в ней выразаем файлы и вставляем на уровень выше, после чего booking1 удаляем;
15. Нажимаем ПКМ на папке с проектом и открываем в IDE. Делаем коммит в новую ветку.
